# 贡献指南

欢迎参与 campus-economy-platform 项目！以下是团队协作和代码贡献的标准流程，适合大一学生小组协作使用。

## 协作流程

### 第一步：查看或创建 Issue

- 在仓库的 Issues 页面查看当前待办任务
- 如果有想做的功能但没有对应 Issue，可以新建一个 Issue 描述你的想法
- 与组长或团队成员讨论确认任务范围

### 第二步：认领任务

- 在对应 Issue 下评论认领，说明你准备完成该任务
- 等待组长确认后即可开始开发

### 第三步：创建开发分支

从 `main` 分支创建你自己的开发分支：

```bash
git checkout main
git pull origin main
git checkout -b feature/你的功能名
```

分支命名建议：
- 新功能：`feature/功能描述`
- 修复 Bug：`fix/问题描述`
- 文档更新：`docs/内容描述`

### 第四步：完成修改

- 在本地完成代码开发
- 保持代码风格与项目现有代码一致
- 确保功能完整且逻辑清晰

### 第五步：本地测试

**在微信开发者工具中测试：**

1. 打开微信开发者工具
2. 导入项目（选择仓库根目录）
3. 使用测试号 AppID
4. 点击「编译」运行
5. 手动测试你修改的功能是否正常

### 第六步：提交 Commit

```bash
git add .
git commit -m "feat: 添加了什么功能"
```

Commit 信息格式建议：
- `feat:` — 新功能
- `fix:` — 修复 Bug
- `docs:` — 文档更新
- `style:` — 代码格式调整
- `refactor:` — 代码重构

### 第七步：发起 Pull Request

- 将你的分支推送到远程仓库
- 在 GitHub 上发起 Pull Request（PR）
- 在 PR 描述中**说明你完成了什么**，如有截图可以附上
- 关联对应的 Issue（如 `Closes #1`）

### 第八步：Code Review

- 由**组长或其他成员** Review 你的代码
- 根据 Review 意见修改代码
- Review 通过后由组长合并

### 第九步：合并到 main

- 合并前请确认：**项目能在微信开发者工具中正常打开并编译运行**
- 合并后删除你的开发分支（可在 PR 页面勾选自动删除）

## 注意事项

- ❌ **不建议直接修改 `main` 分支**，始终通过分支开发 + PR 合并
- ✅ 每次 Pull Request 要写清楚完成了什么功能
- ✅ 合并前至少确认项目能在微信开发者工具打开并编译运行
- ✅ 遇到问题及时在 Issue 或群里沟通
- ✅ 提交前检查是否有调试代码或临时文件残留

## 快速开始

```bash
# 克隆仓库
git clone <仓库地址>
cd campus-economy-platform

# 安装后端依赖
cd server
npm install
cd ..

# 用微信开发者工具打开项目根目录即可
```
