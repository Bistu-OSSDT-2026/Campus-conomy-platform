# CampusMarket 项目运行说明

## 1. 项目简介

CampusMarket 是一个面向高校场景的校园经济平台小程序，主要用于校园二手交易、资料共享、跑腿代取、失物招领和校园服务展示。

本项目当前定位为课程实践与课堂展示版本，重点是展示完整的软件功能思路和开源项目建设过程。后续可继续完善支付功能、接入学校官方系统，并升级为真正服务学生的校园平台。

## 2. 项目结构

建议项目结构如下：

```text
CampusMarket/
├─ miniprogram/        微信小程序源码
├─ server/             后端接口或 mock 服务
├─ web-preview/        网页预览版，用于课堂展示
├─ docs/               项目文档
├─ README.md           项目总说明
├─ LICENSE             开源许可证
└─ 项目运行说明.md
```

如果实际文件夹名称不同，请以实际项目目录为准。

## 3. 微信小程序运行方式

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 选择本项目中的微信小程序源码目录。
4. 填写小程序 AppID；如果只是本地调试，也可以先使用测试号。
5. 点击“编译”，查看首页是否正常显示。
6. 依次测试首页、分类、发布、招领、我的等核心页面。

如果出现代码包过大问题，需要先删除无关大文件、压缩图片资源，或将展示图片改为网络链接。

## 4. Preview 网页运行方式

Preview 网页是为了课堂展示单独生成的浏览器预览版，作用是让同学通过网址快速查看项目效果。

本地运行方式：

```text
打开 web-preview/preview.html
```

或使用 VS Code Live Server：

```text
http://127.0.0.1:5500/preview.html
```

注意：`127.0.0.1` 和 `localhost` 只能在本机访问，别人电脑和手机无法直接打开。

如果要让同学通过网址访问，需要部署到：

- GitHub Pages
- Vercel
- Netlify
- 学长或团队服务器

## 5. 后端或接口说明

课堂展示版本可以使用 mock 数据，不强制依赖真实数据库。

如果后端接口暂时不可用，前端页面应自动使用 mock 数据兜底，避免页面空白。

建议接口包括：

- 商品列表接口
- 商品详情接口
- 发布商品接口
- 收藏接口
- 举报接口
- 失物招领接口
- 用户信息接口

## 6. 核心功能检查

运行前建议检查以下功能：

- 首页是否正常显示
- 分类页面是否能切换
- 商品详情页是否能打开
- 发布页面是否能模拟发布
- 失物招领页面是否能查看和认领
- 我的页面是否能展示用户信息
- Preview 网页是否能打开
- 所有主要按钮是否有反馈

## 7. 常见问题

### 7.1 页面空白

可能原因：

- `preview.html` 路径错误
- JavaScript 报错
- 接口请求失败后没有 mock 数据兜底
- CSS 将内容隐藏

处理方式：

- 检查浏览器控制台报错
- 确认 `preview.html` 文件存在
- 暂时关闭有问题的接口请求
- 使用 mock 数据保证页面先能显示

### 7.2 别人打不开 Preview

如果地址是：

```text
http://127.0.0.1:5500/preview.html
```

这是本机地址，别人无法访问。

解决方式：

- 部署到 GitHub Pages
- 或将本机服务监听地址改为局域网 IP
- 或使用 Vercel、Netlify 等平台部署

### 7.3 微信开发者工具预览失败

如果提示代码包超过限制，需要：

- 删除无用图片、视频、压缩包、文档
- 压缩图片资源
- 移除无关测试文件
- 使用网络图片或云端存储
- 只保留课堂展示所需核心页面

## 8. 当前版本定位

当前版本主要用于课程实践、开源展示和课堂答辩。项目重点不是直接商业上线，而是完成：

- 可运行的软件原型
- 可展示的 Preview 页面
- 可说明的开源策略
- 可扩展的校园服务平台思路

后续可继续完善支付、校园认证、学校官方系统接入、审核后台和真实运营能力。
