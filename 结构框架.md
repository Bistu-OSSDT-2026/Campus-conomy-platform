## 5. 核心模块划分

### 5.1 数据层（Data Layer）

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `defaultProducts[]` | Array | 12条商品数据 |
| `lostFoundItems[]` | Array | 6条失物招领数据 |
| `categories[]` | Array | 7大商品分类 |
| `services[]` | Array | 8个首页服务入口 |
| `campusSvcs[]` | Array | 8个校园服务子项 |
| `notices[]` | Array | 5条校园公告 |
| `myOrders[]` | Array | 4条订单数据 |

### 5.2 状态管理层（State）

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `currentTab` | String | 当前激活的Tab |
| `currentPage` | String | 当前页面ID |
| `pageStack[]` | Array | 页面导航栈（支持返回） |
| `userPosts[]` | Array | 用户发布的商品列表 |
| `favorites[]` | Array | 收藏的商品ID列表 |
| `reports[]` | Array | 举报/认领记录 |

### 5.3 路由系统（Router）

| 函数 | 说明 |
|------|------|
| `navTo(pageId, data)` | 导航到子页面，压栈 |
| `goBack()` | 返回上一页，出栈 |
| `switchTab(tab)` | 切换底部TabBar |
| `switchPage()` | 统一页面切换逻辑 |

### 5.4 页面渲染层（Renderer）

| 函数 | 说明 |
|------|------|
| `renderPage(pageId, data)` | 路由分发器 |
| `renderHome()` | 首页：Banner + 服务入口 + 商品 + 公告 |
| `renderCategory()` | 分类页：分类网格 + 商品列表 |
| `renderDetail(data)` | 商品详情：图片 + 价格 + 描述 + 卖家 |
| `renderLostFound()` | 失物招领：筛选 + 列表 |
| `renderMyOrders()` | 我的订单：订单卡片列表 |
| `renderCampusServices()` | 校园服务：8项服务菜单 |

### 5.5 业务逻辑层（Logic）

| 函数 | 说明 |
|------|------|
| `doSearch()` | 关键词搜索商品 |
| `doPublish()` | 发布商品到 userPosts |
| `toggleFav(id)` | 收藏/取消收藏 |
| `claimItem(id)` | 认领失物，修改状态 |
| `doVerify()` | 学生认证提交 |

### 5.6 UI 工具层

| 函数 | 说明 |
|------|------|
| `showToast(msg)` | 轻提示，1.8秒自动消失 |
| `buildProductCard(p)` | 构建商品卡片 HTML |
| `updateStats()` | 刷新我的页面统计数据 |

## 6. 页面导航流程

```
TabBar 点击
    │
    ├── home  ─────────→ page-home（首页）
    ├── category ──────→ page-category（分类）
    ├── publish ───────→ page-publish（发布）
    ├── lost-found ─────→ page-lost-found（招领）
    └── mine ───────────→ page-mine（我的）

内部导航 navTo()
    │
    ├── 商品卡片 ────→ page-detail ────→ 收藏 / 联系 / 举报
    ├── 订单卡片 ────→ page-my-orders ──→ page-order-detail
    ├── 服务入口 ────→ page-cs ─────────→ 8个服务子页面
    └── 我的菜单 ────→ page-my-posts / favorites / reports 等
```

## 7. 数据流

```
用户操作（点击/输入）
        ↓
事件触发（onclick / oninput）
        ↓
业务逻辑修改状态数组
（userPosts / favorites / reports）
        ↓
DOM 重新渲染（innerHTML）
        ↓
UI 更新 + Toast 反馈
```

## 9. 设计特点

- **零框架**：纯原生实现，无 Vue/React/jQuery 等依赖
- **单文件**：所有 HTML/CSS/JS 集中在一个文件，便于分发
- **页面栈**：类似微信小程序 `wx.navigateTo`，支持多级返回
- **状态隔离**：Tab 切换清栈，子页面导航压栈
- **实时反馈**：所有操作均有 Toast 提示
- **响应式**：固定 390px 手机宽度，桌面端居中显示
- **易扩展**：新增页面只需加一个 `div.page` + 一个 `renderXxx()` 函数

## 10. 使用说明

### 启动预览服务

```bash
cd /path/to/workspace
python -m http.server 8088
```

访问：`http://127.0.0.1:8088/preview.html`

### 快速验证功能

| 操作 | 验证方式 |
|------|----------|
| 首页浏览 | 打开即见 Banner + 服务 + 商品 |
| 搜索 | 顶部搜索框输入"教材" |
| 商品详情 | 点击任意商品卡片 |
| 发布 | Tab「发布」→ 填写表单 → 发布 |
| 失物招领 | Tab「招领」→ 点进详情 → 认领 |
| 校园服务 | 我的 → 校园服务 → 任一服务 |
| 我的页面 | 所有菜单均可点击进入 |
