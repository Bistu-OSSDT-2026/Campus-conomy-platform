# 校园经济平台微信小程序

## 一、项目简介

校园经济平台微信小程序是一个面向高校学生的校园资源流通平台，主要用于校园二手交易、学习资料共享和基础校园服务信息展示。

本项目以北京信息科技大学在校学生为主要用户群体，尝试解决校园内二手物品、学习资料和校园服务信息分散的问题。传统方式通常依赖微信群、QQ群或朋友圈，信息容易被刷屏，也不方便分类查找和长期保存。本项目希望通过一个统一的小程序平台，提高校内资源流通效率。

当前版本为课程实践阶段的基础演示版本，重点不是追求功能数量和技术复杂度，而是保证一个核心使用流程能够完整展示，并完成一次开源项目建设与协作流程实践。

---

## 二、项目定位

本项目定位为“校园经济服务平台”的基础版，主要包括：

- 校园二手物品展示；
- 学习资料和资源共享；
- 信息发布与我的发布管理；
- 校园认证、举报、隐私设置等基础入口；
- 面向课程实践的开源项目文档和协作流程。

当前版本重点展示基础功能和项目结构，后续可继续扩展真实登录、数据持久化、支付、消息通知、后台审核等能力。

---

## 三、核心使用流程

本项目 v0.1.0 版本优先保证以下核心流程能够完整展示：

```text
进入小程序 → 浏览商品或资源 → 查看详情 → 发布信息 → 在“我的发布”中查看发布记录
```

用户可以在首页浏览校园二手商品、学习资料或服务信息，点击条目后进入详情页查看图片、价格、描述、交易地点和联系方式。如果用户有闲置物品或资料，也可以进入发布页面填写信息并提交，之后在“我的发布”中查看自己发布的内容。

---

## 四、当前版本功能

### 1. 首页展示

- 展示校园资源和二手商品信息；
- 提供分类入口；
- 展示轮播图或公告区域；
- 支持进入商品或资源详情页。

### 2. 商品 / 资源详情

- 展示标题、图片、价格、分类、描述等信息；
- 展示交易地点和联系方式；
- 方便用户了解具体资源内容。

### 3. 发布信息

- 支持填写标题、分类、价格、描述、联系方式等信息；
- 支持发布闲置物品或学习资料；
- 发布后可在“我的发布”中查看。

### 4. 我的发布

- 展示用户发布过的内容；
- 作为个人发布记录的基础展示入口。

### 5. 校园认证、举报和隐私设置

- 提供校园认证入口；
- 提供举报入口；
- 提供头像、昵称等个人信息设置入口；
- 当前版本主要用于页面展示和基础交互演示。

## 五、数据与后端说明

本项目当前版本为课程实践阶段的基础演示版本，后端部分主要采用 Mock 接口和模拟数据，用于支撑小程序页面展示、商品浏览、信息发布等核心流程演示。

当前版本暂未接入真实数据库、真实用户登录鉴权和完整的数据持久化机制。因此，部分用户信息，例如头像、昵称等个人资料，在退出登录、重新进入小程序或重新编译项目后，可能会恢复为默认状态。

该限制属于当前 v0.1.0 基础演示版本的已知问题。后续版本计划逐步补充本地缓存或真实后端数据库支持，使用户信息、发布记录和个人设置能够长期保存。

```text
Campus-economy-platform/
├── README.md
├── LICENSE
├── CONTRIBUTING.md
```

目录说明：

- `images/`：项目图片资源；
- `data/`：本地模拟数据；
- `utils/`：公共工具函数；
- `server/`：Mock 后端接口或模拟服务；
- `docs/`：项目说明、设计说明、Issue 拆分说明、测试记录等文档；
- `app.js`、`app.json`、`app.wxss`：微信小程序全局配置和样式文件；
- `project.config.json`：微信开发者工具项目配置文件。

注意：`node_modules/`、`project.private.config.json` 等本地依赖或个人配置文件不应上传到公开仓库。

## 八、运行说明

### 1. 克隆或下载项目

```bash
git clone https://github.com/Bistu-OSSDT-2026/Campus-economy-platform.git
```

也可以在 GitHub 页面点击 `Code`，选择 `Download ZIP` 下载项目压缩包。

### 2. 使用微信开发者工具打开

1. 打开微信开发者工具；
2. 选择“导入项目”；
3. 选择本项目根目录；
4. 使用测试号或自己的 AppID；
5. 点击编译运行。

### 3. 后端说明

如果需要运行 `server/` 目录下的 Mock 后端接口，请进入 `server/` 目录后安装依赖：

```bash
npm install
```

然后根据 `server/README.md` 或相关说明启动服务。

当前版本也可以主要依靠本地模拟数据完成小程序基础展示。

## 十二、已知问题

当前版本仍存在以下限制：

1. 用户头像、昵称等个人信息暂未完整持久化；
2. Mock 后端主要用于课程演示，不等同于真实生产环境后端；
3. 当前版本暂未接入真实登录、数据库和权限控制；
4. 部分校园服务功能仍以页面入口和功能展示为主。

这些问题将作为后续 Issue 和版本迭代方向继续完善。
